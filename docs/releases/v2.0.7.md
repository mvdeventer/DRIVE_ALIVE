- Baseline repository version normalized for automated releases
- Version metadata synchronized across backend, frontend, packaging, and documentation
- Windows installation, update, and release workflow guides added under `docs/`
- Release automation entrypoint added to `s.bat release`

## Installation And Upgrade

- New PC installation guide: `docs/INSTALL_WINDOWS.md`
- Upgrade guide: `docs/UPDATE_WINDOWS.md`
- Release workflow guide: `docs/RELEASE_WORKFLOW.md`

## Operational Notes

- Run `git status --short` before a real release and start from a clean worktree
- Run `.\s.bat release --minor --dry-run` to preview the next release
- Run Alembic migrations when backend schema changes require it
